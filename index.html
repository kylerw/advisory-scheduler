Advisory Scheduler

Assigns every student to an advisory led by a teacher already on their schedule. Everything runs in your browser — the file never leaves this computer.

How to prepare the spreadsheet (start here each year)

Export one file from the SIS with one row per student to be placed. Header rows at the top are fine — the tool detects and skips them automatically.

ColumnContents
AStudent ID number
BScheduling priority: A, B, or C. This only sets processing order (A students tend to have the fewest class options) — it is not a student attribute.
C–JThe student's teacher codes, one column per period (8 periods), numeric codes only. Leave a period blank if the student has no class then — that student simply has fewer advisory options.
  • Each distinct teacher code in the file = one advisory section. Teachers who need to be combined must already be merged into a single shared code before export.
  • Students who are pre-placed by hand must be left out of the file, and their advisories don't appear in it. Every section in the file starts empty.
  • Include only IDs and codes — no names, grades, or other student information.
  • Save as .xlsx or .csv.

Settings: cap is the hard maximum per section (never exceeded); target is the size the balancer aims for. Defaults: cap 29, target 26.

What you get: a master list (student → section), per-section rosters (printable), and a flagged list naming every row that couldn't be placed and exactly why — including which sections are the bottleneck, so fixes happen in the data, not by guessing.

The guarantee: if a full assignment is mathematically possible within the cap, the tool finds it. Anyone flagged genuinely cannot be placed without changing the data or the cap.
